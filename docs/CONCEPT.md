# Project Concept

## Why (One Sentence)

Extend Microsoft's Agent Governance Toolkit (AGT) for LangChain/LangGraph by adding a bypass-proof, on-dispatch-path enforcement hook (`BaseTool.invoke`/`ToolNode.invoke`) that routes decisions into AGT's policy engine — closing the wrapper-bypass gap (AGT #73) that AGT's callback-level adapter leaves open — with an optional Azure upstream backend.

> **Foundation:** qortara depends on AGT (`agent-governance-toolkit-{core,protocols,integrations}`) for the policy engine, identity, and compliance capability; qortara does **not** reimplement them. See [ADR 0001](adr/0001-agt-foundation-vendoring.md).
>
> **Provenance correction:** AGT has been foundational to this project from inception — *not* a later pivot. The genesis bootstrap mischaracterized qortara as an "independent alternative to AGT"; that was an analyst error, corrected in Phase 07. Where ledger/process artifacts say "AGT pivot," read it as "correction of the genesis framing," not a change of project direction.

## Vibe (Three Keywords)

1. **Fail-closed** — uncertainty denies; the system never silently allows when policy evaluation is unavailable.
2. **On-path** — enforcement lives on `BaseTool.invoke` / `ToolNode.invoke`, not on observation hooks.
3. **Inspectable** — every decision is deterministic, reasoned, and produces verifiable evidence.

## Anti-Goals (What This Is NOT)

- Not a reimplementation of AGT — the policy engine, identity, sandboxing, SRE, and OWASP-compliance capability come from AGT as a dependency; qortara adds only the dispatch-path hook + Azure upstream on top.
- Not a complete enterprise governance platform, compliance-reporting suite, or multi-tenant evidence ledger.
- Not a general-purpose policy-authoring UI.
- Not a guarantee that hostile in-process code (which can restore patched methods or call tool functions directly) is contained — the trust boundary is a cooperative application process.
- Not proof that *every* possible framework execution path is governed; only explicitly enumerated, conformance-tested dispatch paths are covered.
- Not (for the current product boundary) a CrewAI / LlamaIndex / AutoGen adapter — the narrow LangChain/LangGraph path is completed first.
- Not a production SLA commitment.
